# AGENTS.md — stratatools

## Commands

```bash
# Go-native dev commands (development-only, replaces st-bootstrap and st-setup)
cd ../guardian && go run ./cmd/guardianctl/ dev init       # full dev init (build + load + deploy)
cd ../guardian && go run ./cmd/guardianctl/ dev setup      # check tools, create kind cluster, generate key
cd ../guardian && go run ./cmd/guardianctl/ dev build      # build images
cd ../guardian && go run ./cmd/guardianctl/ dev deploy     # deploy to K8s
cd ../guardian && go run ./cmd/guardianctl/ dev stop       # scale to zero
cd ../guardian && go run ./cmd/guardianctl/ dev destroy    # delete namespaces
cd ../guardian && go run ./cmd/guardianctl/ dev status     # health check
cd ../guardian && go run ./cmd/guardianctl/ dev ports      # port-forward
cd ../guardian && go run ./cmd/guardianctl/ dev stamp-urls # stamp resolved URLs into partition configs

# Go-native release (replaces st-release)
cd ../guardian && go run ./cmd/guardianctl/ release run --all --bump
cd ../guardian && go run ./cmd/guardianctl/ release run --partition doctor --bump --wait

# Python st-image (still used for image builds)
uv run st-image build --partition guardian-configs
uv run st-image push --partition guardian-configs
uv run st-image stamp --partition guardian-configs

# Go-native dogfood (replaces st-dogfood)
cd ../monofs && go run ./cmd/monofs-admin/ dogfood --router localhost:9090

# Python st-aws-setup
uv run st-aws-setup --aws-profile admin-prod --aws-default-region us-east-1

# Run Python tests:
uv run python -m unittest discover -s tests
```

## Architecture

- **Go CLIs** (primary): `guardianctl` in `~/strata/guardian/` and `monofs-admin` in `~/strata/monofs/`
  - `guardianctl dev` — development-only commands: build, deploy, setup, etc. (replaces st-bootstrap/st-setup)
  - `guardianctl release` — thin orchestrator: image build → stamp → partition push (replaces st-release)
  - `monofs-admin dogfood` — ingest local repos into MonoFS (replaces st-dogfood)
- **Python CLI** (secondary): `st-image` for image building, `st-aws-setup` for IAM provisioning
  - Code in `src/stratatools/image/`, `src/stratatools/aws_setup/`, `src/stratatools/setup/`
- **Templates** in `deploy/bootstrap/` — plain YAML with `${VAR:-default}` envsubst syntax
  - `deploy/bootstrap/storage/` — 27 K8s manifests for MonoFS + MinIO + LB
  - `deploy/bootstrap/guardian/` — 10 K8s manifests for Guardiand + pushers
  - `deploy/bootstrap/bootstrap.yaml` — configuration with defaults
- **Sibling repos**: expects `guardian`, `monofs`, `lb`, `doctor`, etc. alongside stratatools
- **Partitions** live in `partitions/<name>/intents/` — Guardian's desired-state YAML

## Dev flow

1. `guardianctl dev setup` — clone repos, check tools, create kind cluster, generate encryption key
2. `guardianctl dev init` — build Docker images → load into kind → apply K8s templates → wait for readiness
3. `guardianctl dev stamp-urls` — resolve external endpoints → stamp partition configs
4. `guardianctl release run --all --bump` — build partition images → push to registry → guardianctl push/reconcile

## Encryption key

`MONOFS_ENCRYPTION_KEY` (64 hex chars) is generated by `guardianctl dev setup` into `../monofs/.env`.
Never rotate after ingesting data — existing blob archives become unreadable.

## Testing

- Go tests: `cd ../guardian && go test ./cmd/guardianctl/`
- Python tests: `uv run python -m unittest discover -s tests`

## Styling conventions

- `from __future__ import annotations` in every Python file
- Module-level constants for env defaults (UPPER_CASE)
- Internal helpers prefixed with `_`
- `info()` / `warn()` / `die()` for output (from `stratatools.util`)
- `run()` helper logs `+ <cmd>` to stderr, passes `dry_run` / `cwd` / `check`
- Python 3.11 only (`.python-version`)
